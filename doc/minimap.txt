INTRODUCTION					*minimap*

minimap.vim is a plugin that adds minimap to Vim by using Vim 9.2's
|popup-image| feature.

Requires Vim 9.2 or later with |+image| support.


==============================================================================
COMMANDS					*minimap-commands*

:MinimapOpen							*:MinimapOpen*
			Opens minimap for the current window.

:MinimapClose							*:MinimapClose*
			Closes minimap for the current window.


==============================================================================
OPTIONS						*minimap-options*

Every options are containd in *g:minimap_config* dictionary. The default value
will be merged with that dictionary on loading this plugin.

					*g:minimap_config.width*
width			number (default: 10)
			Width of the minimap by number of columns.

					*g:minimap_config.point_height*
point_height		number (default: 1)
			Height of the dots that indicates each character in
			the minimap.

					*g:minimap_config.point_width*
point_width		number (default: 1)
			Width of the dots that indicates each character in the
			minimap.

					*g:minimap_config.line_space*
line_space		number (default: 1)
			Height of the space between each line in the minimap.

					*g:minimap_config.frame_width*
frame_width		number (default: 1)
			Width of the frame to indicate the visible area of
			minimap. If set to 0, this feature is disabled.

					*g:minimap_config.hide_time*
hide_time		number (default: 1000)
			When set to a number greater than 0, minimap will be
			hidden after the specified time (in milliseconds) of
			inactivity. It will be shown again occasionally. If
			set to 0, minimap will not be hidden automatically.
			You can call w:minimap.Hide() and w:minimap.Show() to
			hide/show the minimap manually.

					*g:minimap_config.colors*
colors			dict
			Dictionary of colors for minimap. Each "color code"
			can be string of "#rrggbb", "#rrggbbaa" or color's
			name from |v:colornames|.
		    base	string or dict<string> (default: "Pmenu")
				If it's a string, colors from that highlight
				group will be used to draw minimap.
				If it's a dictionary, it should have color
				codes for "bg" and "fg". Example: >
				{'bg': '#333333', 'fg': '#cccccc'}
<
		    window	string or dict<string> (default: "")
				If it's a string, colors from that highlight
				group will be used to highlight the visible
				area of minimap.
				If it's a dictionary, it should have color
				codes for "bg" and "fg". Example: >
				{'bg': '#333333', 'fg': '#cccccc'}
<				If it is an empty string (default), this
				feature is disabled.

		    frame	string (default: "#ff0000")
				Color code for the frame to indicate the
				visible area of minimap. If it's invalid, this
				feature is disabled.

		    fold	string or dict<string> (default: "")
				Similar to "window" option, but used to
				highlight the folds in the minimap.

					*g:minimap_config.popup_options*
popup_options		dict (default: #{zindex: 10})
			Dictionary of additional options used when creating
			|popup-image| for minimap.
			See |popup_create-arguments| for details.

					*g:minimap_config.cell_pixels*
cell_pixels		list<number> (default: [])
			List of two numbers that are cell width and height in
			pixels. minimap.vim retrieves this value from
			|getcellpixels()| automatically, but you can set this
			manually if there is some problem.

MAYBE FAQ					*minimap-faq*

------------------------------------------------------------------------------
Q:	How to open minimap automatically when opening a file?

A:	This plugin does not provide features to open minimap automatically
	because there are many situations where you don't want that. You can
	make your own |autocommands| to open minimap automatically like:
>
	autocmd VimEnter,WinNew * MinimapOpen
<
